# 写作功能常见问题 (Writing FAQ)

> 面向写作小白的完整指南，不会写文章也能轻松上手！

---

## 📝 什么是写作功能？

**简单理解**：你只需要提供一个观点或想法，AI 会自动帮你扩展成一篇完整的文章。

### 比如这样：

**你只需要说**：
```
我觉得自律是个伪命题
```

**AI 会帮你生成**：
- 完整的文章结构（开头、正文、结尾）
- 丰富的内容扩展
- 精彩的金句
- 匹配的封面提示词

### 和普通 AI 写作有什么不同？

| 普通写作 | writer 写作功能 |
|---------|-------------------|
| 需要详细描述要求 | 只需输入观点 |
| 生成的文章千篇一律 | 有特定创作者风格 |
| 需要手动调整格式 | 直接生成微信格式 |
| 封面需要另外找 | 自动生成封面提示词 |

## 🚀 快速开始

### 方式一：命令行使用

```bash
# 1. 查看所有可用风格
writer write --list

# 2. 交互式写作（最简单）
writer write

# 3. 指定风格写作
writer write --style dan-koe

# 4. 只生成封面提示词
writer write --style dan-koe --cover-only
```

### 方式二：在 Claude Code 中使用

直接用自然语言描述：

```
"用 Dan Koe 风格写一篇关于自律的文章"
"帮我润色这篇文章，让它更有深度"
"生成一个匹配的封面"
```

## 🎨 写作风格是什么？

### 内置风格：Dan Koe

**特点**：
- 深刻但不晦涩
- 犀利但不刻薄
- 有哲学深度但接地气

**适合写什么**：
- 个人成长类文章
- 观点评述
- 人生感悟
- 方法论分享

**写作特点示例**：

| 普通表达 | Dan Koe 风格 |
|---------|-------------|
| 你要努力工作 | 你大概率会放弃你的新年目标，没关系，大多数人都会 |
| 自律很重要 | 自律是个谎言——真正厉害的人根本不需要它 |
| 要多尝试 | 在这个时代，多元兴趣是超能力，不是缺陷 |

### 如何添加自己喜欢的风格？

1. 参考内置风格文件 `writers/dan-koe.yaml`
2. 创建自己的风格文件 `writers/my-style.yaml`
3. 填写风格描述和写作提示词
4. 立即可用！

详细教程见 [writers/README.md](../writers/README.md)

## 📋 输入类型说明

### 观点/想法 (idea)

**最简单的输入方式**，只需一句话表达你的观点。

**示例**：
```
我觉得年轻人都不爱读书了
拖延症不是懒，是恐惧
```

### 内容片段 (fragment)

**已有部分内容**，需要扩展成完整文章。

**示例**：
```
最近发现一个现象：身边的朋友都在谈论"躺平"。
有人说这是年轻人不努力，但我觉得不是这样...
```

### 大纲 (outline)

**有结构思路**，需要填充内容。

**示例**：
```
标题：为什么我不再做新年计划

1. 新年计划的失败率很高
2. 问题不在于执行力
3. 关键在于认知升级
4. 更好的替代方案
```

### 标题扩展 (title)

**只有一个标题**，需要围绕标题写文章。

**示例**：
```
自律是个谎言
```

## 🔧 常见问题

### Q1: 我完全不会写文章，能用吗？

**A: 完全可以！** 这就是为小白设计的：

1. **只需要有想法**：哪怕只有一个观点
2. **AI 自动扩展**：会生成完整的文章结构
3. **风格加持**：自动应用专业写作技巧

**试试这样**：
```bash
writer write
# 然后输入：我觉得自律是个伪命题
```

### Q2: 生成的内容可以直接发公众号吗？

**A:** 生成的是 Markdown 格式，需要先转换为微信格式：

```bash
# 1. 生成文章（保存到文件）
writer write --style dan-koe -o article.md

# 2. 转换为微信格式
writer convert article.md --preview

# 3. 满意后推送草稿
writer convert article.md --draft --cover cover.jpg
```

### Q3: 可以修改生成的内容吗？

**A: 当然可以！** 生成的文章是起点，不是终点：

- 直接修改 Markdown 文件
- 或者用 `--input-type fragment` 重新润色

### Q4: 封面怎么用？

**A:** 封面生成的是**图片提示词**：

```bash
# 生成封面提示词
writer write --style dan-koe --cover-only
```

然后：
1. 复制生成的提示词
2. 用 AI 图片生成工具生成图片（**推荐使用 16:9 比例**）
3. 使用 `writer image upload` 上传到微信
4. 在转换时用 `--cover` 指定封面图

**公众号封面图尺寸建议**：

| 用途 | 推荐尺寸 | 说明 |
|------|----------|------|
| **文章封面** | 2560x1440 (16:9) | 横向比例，在微信 feed 流和文章列表显示效果更好 |
| **默认生成** | 2048x2048 (1:1) | 方形图片，在预览时会被裁剪 |

```bash
# 生成 16:9 比例的封面图（推荐）
writer generate_image --size 2560x1440 "你的封面提示词"

# 使用封面图推送草稿
writer convert article.md --draft --cover cover.jpg
```

### Q5: 如何提高生成质量？

**A:** 提供更多上下文：

**差**：
```
写一篇关于自律的文章
```

**好**：
```
我觉得自律是个伪命题。那些真正厉害的人，
从来不觉得自己在"坚持"什么，他们只是没办法不那样做。
```

**技巧**：
- 分享个人经历
- 提供具体例子
- 说明想要的观点

### Q6: 支持中文吗？

**A: 完全支持！** 专为中文写作优化：

- 内置风格都是中文创作者风格
- 生成的文章符合中文表达习惯
- 金句、引用都是中文语境

### Q7: 生成的文章有版权问题吗？

**A:** 内容由 AI 根据你的观点生成，版权属于你。

但建议：
- 始终基于自己的观点
- 适当修改生成内容
- 加入个人经历和案例

### Q8: 如何添加自己的风格？

**A:** 创建 YAML 配置文件：

```bash
# 1. 查看内置风格示例
cat writers/dan-koe.yaml

# 2. 创建自己的风格
vim writers/my-style.yaml
```

**最简单的配置**：
```yaml
name: "我的风格"
english_name: "my-style"
description: "简洁有力"

writing_prompt: |
  你是一位简洁有力的写作者。
  用最少的字表达最清晰的观点。
  避免废话，直击要点。
```

详细教程：[writers/README.md](../writers/README.md)

## 💡 使用技巧

### 技巧1：从小观点开始

不要一上来就想写大文章，从小观点入手：

```
✅ 好的开始：
- 我觉得年轻人不爱读书了
- 拖延症不是懒
- 社交媒体让人焦虑

❌ 太宽泛：
- 写一篇关于教育的文章
- 谈谈人工智能
```

### 技巧2：多次迭代

第一次生成不满意？试试：

```bash
# 第一次：用观点生成
writer write --style dan-koe

# 第二次：把生成的内容重新润色
writer write --style dan-koe --input-type fragment article.md
```

### 技巧3：提取金句

生成文章后，提取金句发朋友圈：

```bash
# 金句会自动标注
=== 金句 ===
1. 自律是个谎言——真正厉害的人根本不需要它
2. 你不是缺乏方向，你是方向太多
```

### 技巧4：组合使用

```bash
# 1. 写文章
writer write --style dan-koe -o article.md

# 2. 生成封面
writer write --style dan-koe --cover-only > cover_prompt.txt

# 3. 转换微信格式
writer convert article.md --mode ai --theme autumn-warm --preview
```

## 🎯 实战案例

### 案例1：观点 → 文章

**输入**：
```
我觉得年轻人都不爱读书了，但这不是问题。
```

- 标题：《为什么"不爱读书"不是问题》
- 结构：钩子 → 痛点共鸣 → 核心观点 → 行动指南

### 案例2：片段 → 润色

**输入**：
```
最近发现一个现象：身边的朋友都在谈论"躺平"。
有人说这是年轻人不努力，但我觉得不是这样...
```

**生成结果**：
- 保持原有观点
- 用 Dan Koe 风格重新表达
- 补充论证和例子
- 提炼核心金句

### 案例3：大纲 → 完整文章

```
标题：为什么我不再做新年计划

1. 新年计划的失败率很高
2. 问题不在于执行力
3. 关键在于认知升级
```

**生成结果**：
- 按大纲结构填充内容

- 开头有钩子，结尾有行动指南
